## Dual-Table Pattern
- Insert into `users` (auth: login, JWT, password reset tokens)
- Insert into `memberProfiles` (admin panel visibility)
- Both share email as linking key

## Error Handling
- 400: missing name/email, invalid role
- 401: not authenticated
- 403: not admin
- 409: email already exists (checked in both tables)
- 503: database unavailable
- Email fail: 201 + warning (no rollback — admin can resend)
