# TASK-0131 — Dashboard Input Contrast Fix (Systemic)

**Status:** done
**Priority:** high
**Sprint:** 2026-05-15

## Root Cause
`body { color: var(--text-primary) }` = `#eaeaea` (landing dark theme).
Dashboard uses `bg-white` but all `<input>`, `<textarea>`, `<select>` inherit `#eaeaea` = invisible text on white.

This has been "fixed" per-tool multiple times (yamaq) but keeps returning because new tools inherit body color.

## Fix
`app/globals.css` — global scope, not per-tool:
```css
input, textarea, select { color: var(--dk-ink); }
input::placeholder, textarea::placeholder { color: var(--dk-ink-soft); opacity: 1; }
```

## Impact
All 11 marketing tools + all dashboard forms. No per-component changes needed.
