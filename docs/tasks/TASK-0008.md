# TASK-0008 — Resend Email Integration

**Status:** Done
**Branch:** feat/resend-email
**Date:** 2026-04-12

## Məqsəd

`sendEmail` placeholder console.log-larını Resend üzərindən real email göndəriminə çevirmək.

## Edilənlər

### Yeni fayllar
- `lib/email/client.ts` — Resend singleton, `canSendEmail()`, `EMAIL_FROM`, `ADMIN_EMAIL`
- `lib/email/send.ts` — `sendEmail({ to, subject, html })` universal funksiya
- `lib/email/templates/new-lead.ts` — Admin bildirişi: yeni müraciət
- `lib/email/templates/lead-confirmation.ts` — Müraciət edənə avtomatik təsdiq
- `lib/email/templates/listing-approved.ts` — Sahibkara: elan vitrinə çıxdı

### Dəyişdirilən fayllar
- `lib/email/templates.ts` — `sendEmail()` indi `send.ts`-ə yönlənir (console.log silindi)
- `app/api/listings/[id]/leads/route.ts` — Lead POST: admin + lead confirmation email
- `app/api/listings/[id]/status/route.ts` — Status PATCH: `showcase_ready` → sahibkara email
- `lib/members/auth-adapter.ts` — Qeydiyyat: email verification real göndərilir
- `.env.example` — `RESEND_API_KEY`, `EMAIL_FROM`, `ADMIN_EMAIL`, `NEXT_PUBLIC_SITE_URL` əlavə edildi
- `docs/ENV-SETUP.md` — Resend bölməsi əlavə edildi

## Email Axını

```
Lead submit (POST /api/listings/[id]/leads)
  → Admin bildirişi (ADMIN_EMAIL)
  → Lead confirmation (lead.email — varsa)

Status change → showcase_ready (PATCH /api/listings/[id]/status)
  → Sahibkara elan təsdiq emaili (listing.email — varsa)

Qeydiyyat (lib/members/auth-adapter.ts)
  → Email verification (member email-i)
```

## Fallback

`RESEND_API_KEY` yoxdursa: `console.warn` yazılır, build qırılmır, API cavabı dəyişmir.
