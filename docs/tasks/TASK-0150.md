- `components/marketinq-ocagi/trend-analiz/TrendAnalizPage.tsx` - single client component.

- `messages/{az,en,ru,tr}.json` - `marketinq.trendAnaliz` namespace.

## Architecture Decision

- RSS is intentionally out of scope.
- Static trend KB keeps the tool stable and sprint-safe.
- AI is only used to translate the top 3 trends into a concrete first step for the user's restaurant profile.
- If AI fails, the component shows static `fallbackFirstStep` copy and an AI-unavailable note.

## Trend KB

1. Value and price sensitivity
2. Delivery-first format
3. AI and digital ordering
4. Health and functional menu
5. Nostalgia and flavor escape
6. Beverage focus and margin
7. Sustainability and local sourcing
8. Experience and human touch

## Scoring

Each trend has a restaurant-type fit, audience fit, and current-strength fit.

Formula:

```text
score = restaurantType * 0.5 + audience * 0.3 + strength * 0.2
```

The highest 3 trends become the priority cards.
